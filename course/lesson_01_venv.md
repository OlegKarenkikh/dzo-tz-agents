# 🖥️ Урок 1: Изолированное окружение (venv)

![venv illustration](images/lesson_01_venv.png)

---

## 🤔 Что такое «изолированное окружение»?

Представьте, что ваш компьютер — это большой офис, а каждый Python-проект — это отдельный стол.
В одном проекте нужна версия библиотеки 1.0, в другом — 2.0.
Без изоляции они будут мешать друг другу. **venv** создаёт для каждого проекта свою отдельную папку с библиотеками.

```
Без venv:          С venv:

[Компьютер]        [Компьютер]
    |                  |
[Python 3.11]     [Python 3.11]
    |               ┌───────┐   ┌───────┐
[langchain 1.0]   |проект 1| |проект 2|
    |             |вер. 1.0| |вер. 2.0|
[pydantic 2.0]    └───────┘   └───────┘
«конфликт версий!>   «каждый чист — всё работает!»
```

---

## 📦 Как это устроено в dzo-tz-agents?

В файле [requirements.txt](../requirements.txt) сохранены **все зависимости** проекта:

```txt
langgraph>=0.2
langchain-openai>=0.1
fastapi>=0.111
streamlit>=1.35
pdfplumber>=0.11
...
```

Когда вы запускаете `pip install -r requirements.txt` —
Python загружает именно эти библиотеки в ваше виртуальное окружение (venv),
не затрагивая остальные проекты на вашем компьютере.

---

## ✅ Практика: запускаем проект в изолированном окружении

### Шаг 1: Склонировать репозиторий

```bash
git clone https://github.com/OlegKarenkikh/dzo-tz-agents.git
cd dzo-tz-agents
```

> 💡 `git clone` — это как «скачать» проект с GitHub на ваш компьютер.

### Шаг 2: Создать виртуальное окружение

```bash
python -m venv .venv
```

Это создаст папку `.venv` внутри проекта — наш «отдельный стол».

### Шаг 3: Активировать окружение

```bash
# macOS / Linux:
source .venv/bin/activate

# Windows (PowerShell):
.venv\Scripts\Activate.ps1
```

После этого в начале строки терминала появится:

```
(.venv) ← это значит, что вы внутри!
(.venv) $ _
```

### Шаг 4: Установить библиотеки

```bash
pip install -e ".[ui,dev]"
```

> 💡 `-e` — это «редактируемый режим»: изменения в коде сразу вступают в силу без переустановки.

### Шаг 5: Скопировать конфигурацию

```bash
cp .env.example .env
```

Откройте `.env` в любом текстовом редакторе и заполните свои значения (OPENAI_API_KEY и др.).

## ✨ Проверка: всё работает!

Запустите API:

```bash
make api
```

Или вручную:

```bash
python -m uvicorn api.app:app --host 0.0.0.0 --port 8000
```

Если видите `INFO: Application startup complete.` — всё заработало! 🎉

Откройте в браузере: [http://localhost:8000/docs](http://localhost:8000/docs)
Вы увидите интерактивную документацию Swagger — это все API-методы проекта.

## 📍 Что запомнить

| Понятие | Что значит |
|---|---|
| `venv` | Отдельная папка с библиотеками для проекта |
| `activate` | Войти в окружение |
| `pip install` | Установить библиотеку |
| `.env` | Файл с секретными настройками (не публикуется!) |
| `requirements.txt` | Список зависимостей |
