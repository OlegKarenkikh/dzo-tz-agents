## 🤔 Что такое «изолированное окружение»?

Представьте, что ваш компьютер — это большой офис, а каждый Python-проект — это отдельный стол.
В одном проекте нужна версия библиотеки 1.0, в другом — 2.0.
Без изоляции они будут мешать друг другу. **venv** создаёт для каждого проекта свою отдельную папку с библиотеками.

```
Без venv:             С venv:

[Компьютер]           [Компьютер]
     |                    |
[Python 3.11]        [Python 3.11]
     |               ┌──────────┐  ┌──────────┐
[langchain 1.0]      │проект DZO│  │проект X  │
     |               │venv/     │  │venv/     │
[pydantic 2.0]       │(своя вер)│  │(своя вер)│
     |               └──────────┘  └──────────┘
  КОНФЛИКТ!          «каждый изолирован — всё работает!»
```

## 📦 Что в requirements.txt?

В файле [requirements.txt](../requirements.txt) сохранены **все зависимости** проекта:

```txt
langgraph>=0.2        # фреймворк для агентов
langchain-openai>=0.1 # подключение к GPT-4o
fastapi>=0.111        # веб-сервер для API
streamlit>=1.35       # веб-интерфейс (необязательно)
pdfplumber>=0.11      # чтение PDF-файлов
```

> 💡 `requirements.txt` — это список покупок. `pip install` — это поход в магазин.

## ✅ Практика: запускаем проект

### Шаг 1: Склонировать репозиторий

```bash
git clone https://github.com/OlegKarenkikh/dzo-tz-agents.git
cd dzo-tz-agents
```

> 💡 `git clone` — скачать проект. `cd` — войти в папку. [Подробнее о Git →](lesson_00_terminal.md)

### Шаг 2: Создать виртуальное окружение

```bash
python -m venv .venv
```

Это создаст папку `.venv` — наш «отдельный стол».

### Шаг 3: Активировать окружение

> 💡 **Что такое `source`?**
> `source` — команда bash, которая выполняет скрипт **в текущей оболочке** (не в дочерней).
> Это важно: `activate` меняет переменные окружения текущего терминала.
> Если запустить скрипт обычно (`./activate`), он изменит переменные в дочернем процессе, а текущий терминал не заметит.
> Аналогия: `source` = «прочитать инструкцию вслух», обычный запуск = «дать инструкцию помощнику, который сам в ней разберётся».

```bash
# macOS / Linux:
source .venv/bin/activate

# Windows (PowerShell):
.venv\Scripts\Activate.ps1
```

После этого в начале строки появится:
```
(.venv) $ _  ← вы внутри окружения!
```

### Шаг 4: Установить библиотеки

```bash
pip install -e ".[ui,dev]"
```

> 💡 **Откуда берётся `[ui,dev]`?**
> В корне проекта есть файл `pyproject.toml` — это конфиг пакета Python.
> Там описаны «дополнительные наборы зависимостей»:
> - `[ui]` — Streamlit и другие компоненты веб-интерфейса
> - `[dev]` — инструменты разработки (тесты, линтеры)
> Без `[ui,dev]` установятся только базовые зависимости.

> 💡 Что значит `-e`?
> Обычный `pip install` копирует файлы и «замораживает» их.
> `-e` (editable mode) — устанавливает «ссылку» на папку проекта.
> Это значит: когда вы редактируете код — изменения вступают в силу сразу, без переустановки.

### Шаг 5: Скопировать конфигурацию

> 💡 **`cp` на Windows не работает в cmd!**
> В Git Bash — `cp` работает. В стандартном cmd — используйте `copy`:
> ```cmd
> copy .env.example .env
> ```
> Или создайте `.env` вручную: скопируйте файл `.env.example` через Проводник, переименуйте в `.env`.

```bash
cp .env.example .env
```

Откройте `.env` в редакторе и заполните значения. [Подробнее о токенах →](lesson_04_token.md)

## ✨ Проверка: всё работает!

```bash
make api
```

> 💡 **Что такое `make`?**
> `make` — утилита для запуска команд из файла `Makefile`. Это как «ярлыки» для длинных команд.
> `make api` заменяет длинную команду: `uvicorn api.app:app --reload --host 0.0.0.0 --port 8000`
> На macOS/Linux `make` уже установлен. На Windows: установите через `winget install GnuWin32.Make`
> или через Git Bash (входит в состав Git for Windows).

Если видите `INFO: Application startup complete.` — всё заработало! 🎉

Откройте в браузере: **http://localhost:8000/docs**

Вы увидите **Swagger UI** — это интерактивная документация API.
Там можно нажать на любой метод и отправить запрос прямо из браузера:

```
http://localhost:8000/docs

┌─────────────────────────────────────┐
│ POST /api/v1/dzo/inspect   [Try it] │
│ POST /api/v1/tz/inspect    [Try it] │
│ GET  /api/v1/jobs/{id}     [Try it] │
│ GET  /health               [Try it] │
└─────────────────────────────────────┘
```

## 📍 Что запомнить

| Понятие | Что значит |
|---|---|
| `venv` | Изолированная папка с библиотеками проекта |
| `source .venv/bin/activate` | Войти в окружение |
| `(.venv)` в строке | Подтверждение, что вы внутри окружения |
| `pip install -e` | Установить в редактируемом режиме (удобно для разработки) |
| `.env` | Файл с секретными настройками (не загружается на GitHub!) |
| `requirements.txt` | Список всех нужных библиотек |
| Swagger `/docs` | Интерактивная документация API в браузере |
