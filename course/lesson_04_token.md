# 🔑 Урок 4: Токен — ваш ключ к агентам

---

## 🤔 Что такое токен?

**Токен** — это длинный секретный ключ, который подтверждает вашу личность при обращении к API.
Представьте гостиницу: вам дают карточку-ключ от комнаты. Без неё вы не войдёте.
Токен — это такая же карточка для API.

В проекте dzo-tz-agents используется два вида токенов:
- **API_KEY** — ключ к нашему собственному серверу (FastAPI)
- **OPENAI_API_KEY** — ключ к сервису OpenAI (GPT-4o)

---

## 📦 Где хранятся токены?

Токены хранятся в файле `.env` — он находится в корне проекта.

```
# Файл .env (никогда не публикуйте его в GitHub!)

API_KEY=секретный_ключ_нашего_сервера
OPENAI_API_KEY=sk-proj-...ваш_ключ_openai...

POSTGRES_PASSWORD=пароль_базы_данных
```

> ⚠️ **Важно:** файл `.env` добавлен в `.gitignore` — Git никогда не загрузит его на GitHub.
> Это защита от случайной утечки паролей.

## ✅ Практика: работаем с токенами

### Шаг 1: Создайте файл .env

```bash
cp .env.example .env
```

Откройте `.env` в редакторе:

```bash
nano .env
# или
code .env  # если установлен VS Code
```

### Шаг 2: Заполните ключи

Минимально необходимые ключи для запуска:

```
API_KEY=любой_длинный_случайный_текст
OPENAI_API_KEY=sk-proj-...
```

Хороший API_KEY можно сгенерировать прямо в терминале:

```bash
python3 -c "import secrets; print(secrets.token_hex(32))"
```

### Шаг 3: Используйте токен в curl

```bash
# Замените YOUR_API_KEY на значение из .env → API_KEY
curl http://localhost:8000/api/v1/dzo/inspect \
  -X POST \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{"document": "Заявка от ООО Тест"}'
```

### Шаг 4: Проверьте, что токен работает

```bash
# Без токена — ошибка 401 Unauthorized
curl http://localhost:8000/api/v1/dzo/inspect -X POST \
  -H "Content-Type: application/json" \
  -d '{"document": "тест"}'
# → {"detail": "Not authenticated"}

# С токеном — всё работает
curl http://localhost:8000/api/v1/dzo/inspect -X POST \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{"document": "тест"}'
# → {"job_id": "...", "status": "accepted"}
```

## 🔐 Правила безопасности токенов

| Правило | Почему |
|---|---|
| Никогда не публикуйте токен в GitHub | Боты сканируют репозитории и крадут ключи за минуты |
| Не передавайте токен в чатах | История чата может утечь |
| Используйте разные токены для dev и prod | Если утечёт dev-ключ, prod останется защищённым |
| Регулярно обновляйте токены | На случай незаметной утечки |
| Если токен утёк — сразу отзовите | GitHub: Settings → Developer Settings → Tokens |

---

## 🔄 Как отозвать и заменить токен?

Если вы случайно опубликовали токен:

1. **GitHub токен:** [github.com/settings/tokens](https://github.com/settings/tokens) → Delete
2. **OpenAI токен:** [platform.openai.com/api-keys](https://platform.openai.com/api-keys) → Revoke
3. Создайте новый токен и обновите `.env`
4. Перезапустите сервер: `make api`
