# 🌐 Урок 3: curl — разговариваем с агентом

---

## 🤔 Что такое curl?

**curl** — это программа для отправки запросов по интернету прямо из терминала.
Представьте, что браузер — это машина, а curl — это велосипед: медленнее, зато вы контролируете каждое движение.

Когда вы открываете сайт в браузере — браузер отправляет HTTP-запрос.
curl делает то же самое, но вручную, из командной строки.

---

## 📦 Что такое API?

**API** (Application Programming Interface) — это «меню» программы.
Вы отправляете запрос (заказ) — получаете ответ (блюдо).

В нашем проекте API — это FastAPI-сервер на порту 8000.
Он принимает документы ДЗО/ТЗ и возвращает результат проверки.

---

## ✅ Практика: первые запросы к агенту

### Шаг 1: Убедитесь, что сервер запущен

```bash
make api
```

В другом окне терминала выполните:

```bash
curl http://localhost:8000/health
```

Ответ должен быть:
```json
{"status": "ok"}
```

### Шаг 2: Посмотрите документацию API

Откройте в браузере: [http://localhost:8000/docs](http://localhost:8000/docs)

Swagger UI покажет все доступные методы — можно отправлять запросы прямо из браузера!

### Шаг 3: Отправьте документ ДЗО на проверку

```bash
curl -X POST http://localhost:8000/api/v1/dzo/inspect \
  -H "Content-Type: application/json" \
  -H "X-API-Key: ваш_api_key" \
  -d '{
    "document": "Заявка от ООО Ромашка на страхование КАСКО"
  }'
```

Ответ будет примерно такой:
```json
{
  "job_id": "abc123",
  "status": "accepted"
}
```

### Шаг 4: Проверьте статус задания

```bash
curl http://localhost:8000/api/v1/jobs/abc123 \
  -H "X-API-Key: ваш_api_key"
```

## 🔑 Анатомия curl-запроса

```
curl -X POST            ← метод HTTP (POST, GET, PUT, DELETE)
  http://...            ← адрес сервера
  -H "..."              ← заголовок (header)
  -d '{...}'            ← тело запроса (данные)
```

| Флаг | Что делает |
|---|---|
| `-X POST` | Метод запроса (по умолчанию GET) |
| `-H "Content-Type: application/json"` | Говорим серверу: данные в формате JSON |
| `-H "X-API-Key: ..."` | Передаём токен авторизации |
| `-d '{"key": "val"}'` | Тело запроса с данными |
| `-s` | Silent — не показывает прогресс |

## 📍 Что запомнить

| Термин | Значение |
|---|---|
| `curl` | Утилита для HTTP-запросов из терминала |
| `API` | Интерфейс программы для взаимодействия |
| `GET` | Получить данные |
| `POST` | Отправить данные |
| JSON | Формат данных: `{"ключ": "значение"}` |
| Header | Заголовок запроса (метаданные) |
