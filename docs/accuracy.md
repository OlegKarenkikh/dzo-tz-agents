## Методология

### Тестовая база
Тестовые документы хранятся в `tests/fixtures/real_procurement_docs.py`.
Каждый документ содержит:
- Полный текст (извлечённый из реального документа или синтетический)
- Ground truth (экспертная оценка: решение, отсутствующие разделы, процент заполненности)
- Ожидаемое решение rules-engine
- Источник (URL) и признак верификации

### Источники документов
1. **ЕЭК (Евразийская экономическая комиссия)** — Конкурс №1205, закупка лицензий ПО
   - URL: https://eec.eaeunion.org/upload/iblock/474/s35m15bhwpgshpcwo4ajapf4mgdenqzo/1205.pdf
2. **rbank.by** — ТЗ на закупку компьютерного оборудования (моноблоки, ноутбуки)
   - URL: https://rbank.by/upload/medialibrary/a99/Tekh.zadanie-na-tender.pdf
3. **АО «Клевер»** — ТЗ на поставку листогибочного оборудования
   - URL: https://www.kleverltd.ru/upload/iblock/f02/TZ-na-valtsy-24.06.2025.doc
4. **ПАО «Россети»** — Единый стандарт закупок (223-ФЗ)
   - URL: https://rossetimr.ru/zakupki/prav_obesp/2024/12-27-st-n.pdf
5. Синтетические заявки ДЗО (на основе публичных шаблонов 44-ФЗ/223-ФЗ)

### Два режима тестирования
1. **Rules-engine (без LLM)** — проверяет структурный парсер `detect_sections()`.
   Быстрый, детерминированный, но ограничен keyword-matching.
2. **LLM E2E** — полный цикл через API с реальной моделью.
   Точнее, но требует API-ключ и больше времени.

### Known limitations
- Rules-engine может давать ложноположительные результаты на документах,
  где ключевые слова встречаются в теле характеристик (например, «г.» как единица измерения).
- LLM-режим устраняет это ограничение за счёт семантического анализа.
- Целевая accuracy ≥85% для LLM-режима, ≥70% для rules-engine.

## Запуск тестов точности

```bash
# Rules-engine only (быстро, без LLM)
pytest tests/test_real_procurement_docs.py -v

# E2E с LLM
E2E_API_KEY=your_key E2E_API_BASE=your_base E2E_MODEL_NAME=model \
  pytest tests/test_real_procurement_docs.py tests/test_accuracy_report.py -m e2e -v
```

## Целевые метрики v2.0
- Rules-engine accuracy: ≥70% на всех документах
- LLM E2E accuracy: ≥90% на всех агентах
- Повторяемость: temperature=0.0, seed=42 → идентичные результаты
- Минимум 10 реальных документов в тестовой базе
