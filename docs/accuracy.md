# Точность агентов: измеренные метрики

## Методология

- **Ground truth:** экспертное решение специалиста по госзакупкам
- **Тестовые документы:** реальные публичные источники (ЕЭК, rbank.by) и синтетические шаблоны (44-ФЗ/223-ФЗ)
- **Режимы тестирования:**
  - Rules-engine (без LLM): структурный парсинг + keyword matching
  - LLM E2E: полный цикл через API с реальной моделью (qwen3-32b)

## Результаты (rules-engine, без LLM)

| Документ | Источник | Решение эксперта | Решение rules-engine | Совпадение |
|---|---|---|---|---|
| ЕЭК ТЗ 2024 | [eec.eaeunion.org](https://eec.eaeunion.org/upload/iblock/474/s35m15bhwpgshpcwo4ajapf4mgdenqzo/1205.pdf) | ПРИНЯТЬ С ЗАМЕЧАНИЕМ | ВЕРНУТЬ НА ДОРАБОТКУ | ❌ |
| rbank ТЗ 2021 | [rbank.by](https://rbank.by/upload/medialibrary/a99/Tekh.zadanie-na-tender.pdf) | ВЕРНУТЬ НА ДОРАБОТКУ | ПРИНЯТЬ | ❌ (ложноположительный) |
| Заявка ДЗО ОКЭ-2025 | Синтетический (44-ФЗ/223-ФЗ) | ВЕРНУТЬ НА ДОРАБОТКУ | ВЕРНУТЬ НА ДОРАБОТКУ | ✅ |

**Итог rules-engine:** 1/3 = 33.3% decisions_match

## Целевые показатели

| Режим | Текущая точность | Целевая точность |
|---|---|---|
| Rules-engine (без LLM) | 33.3% | ≥ 70% |
| LLM E2E (qwen3-32b) | Измеряется | ≥ 85% |

## Известные ограничения rules-engine

1. **Ложноположительный результат для rbank:** keyword `г.` встречается в характеристиках оборудования (размер дисплея "23.8\""), а не в поле адреса. Rules-engine неверно определяет наличие раздела «место поставки».
2. **Строгий порог агента vs эксперт:** Агент использует порог 85% для принятия, эксперт допускает принятие с устным замечанием при 87.5% (ЕЭК).
3. **LLM-режим устраняет оба ограничения** за счёт семантического анализа контекста.

## Запуск тестов точности

```bash
# Rules-engine only (быстро, без LLM)
pytest tests/test_real_procurement_docs.py -v

# E2E с LLM (требует API-ключ и работающий сервер)
QWEN_API_KEY=your_key pytest tests/ -m e2e -v
```

## Файлы с тестовыми данными

- `tests/fixtures/real_procurement_docs.py` — тексты документов и ground truth
- `tests/fixtures/real_procurement_expected.json` — структурированные ожидаемые результаты
